Atividade HTML:

1- HTML, ou HyperText Markup Language, é a linguagem padrão utilizada para criar e estruturar páginas web. Seu papel principal é definir a estrutura e o conteúdo das páginas web.
2- Tags em HTML são usadas para marcar e identificar diferentes partes de um documento web. Por exemplo, a tag <p> é usada para definir parágrafos em um documento HTML.
3- Um elemento em HTML consiste em uma tag de abertura, conteúdo e, em muitos casos, uma tag de fechamento. Por exemplo, <p>Este é um parágrafo</p> é um elemento de parágrafo que contém o texto "Este é um parágrafo".
4- Atributos em HTML fornecem informações adicionais sobre um elemento e são definidos dentro das tags de abertura. Por exemplo, o atributo "href" em um elemento <a> define o link de destino, como em <a href="https://www.example.com">Link Exemplo</a>.
5- A versão mais recente do HTML é o HTML5. Alguns dos recursos avançados suportados por HTML5 incluem elementos de vídeo e áudio nativos, armazenamento local, gráficos vetoriais, entre outros.

Atividades CSS:

1- O propósito principal do CSS é estilizar elementos HTML e controlar sua aparência em uma página da web, separando a estrutura do conteúdo HTML de sua apresentação visual.
2- Alguns dos principais recursos e conceitos do CSS mencionados no texto incluem seletores, propriedades CSS, classes e IDs, Box Model, Flexbox e Grid.
3- Alguns dos desafios enfrentados pelos desenvolvedores ao trabalhar com CSS incluem compatibilidade entre navegadores e desempenho. Esses desafios podem ser mitigados testando em vários navegadores e seguindo as melhores práticas de codificação para melhorar o desempenho.
4- O Box Model no CSS descreve como os elementos HTML são renderizados em uma página, incluindo margens, bordas, preenchimento e conteúdo. É importante entender o Box Model para controlar o layout dos elementos de forma eficaz.
5- Algumas das técnicas de layout avançadas do CSS mencionadas no texto incluem Flexbox e Grid, que permitem criar layouts flexíveis e responsivos de forma eficiente.

Atividades Git:

1- A diferença entre o Git e o GitHub é que o Git é um sistema de controle de versão distribuído, enquanto o GitHub é uma plataforma de hospedagem de código que utiliza o Git para facilitar a colaboração entre desenvolvedores.
2- Alguns dos principais conceitos do Git mencionados no texto incluem repositório, branches, merge e conflitos de merge. Esses conceitos são importantes para rastrear e gerenciar alterações no código fonte de um projeto.
3- Os desafios enfrentados pelos desenvolvedores ao trabalhar com Git incluem resolução de conflitos de merge e colaboração em projetos distribuídos. 4- 4- Esses desafios podem ser mitigados seguindo práticas recomendadas, como manter commits pequenos e frequentes e revisar cuidadosamente as alterações antes de mesclar.
5- O Git e o GitHub revolucionaram a forma como os desenvolvedores colaboram e gerenciam projetos de software, proporcionando uma base sólida para o desenvolvimento de software eficiente e colaborativo.
6- É importante seguir práticas recomendadas ao trabalhar com Git e GitHub para garantir uma colaboração eficaz e um gerenciamento de projeto suave.
